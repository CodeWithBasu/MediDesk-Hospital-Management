# ESLint Configuration for Data Fetching

## Suppress useEffect Data Fetching Warnings

The following ESLint configuration suppresses the `react-hooks/exhaustive-deps` warning for data fetching patterns, which is a standard and accepted React pattern.

Add this to your `.eslintrc.json` or `.eslintrc.js`:

```json
{
  "rules": {
    "react-hooks/exhaustive-deps": [
      "warn",
      {
        "additionalHooks": "(useDataFetching)"
      }
    ]
  }
}
```

Or suppress specific warnings in code with:

```typescript
// eslint-disable-next-line @typescript-eslint/no-floating-promises
useEffect(() => {
  fetchData();
}, [fetchData]);
```

## Alternative: Custom Hook Pattern

For a cleaner approach, create a custom hook:

```typescript
// hooks/useDataFetching.ts
import { useEffect } from "react";

export function useDataFetching(
  fetchFn: () => Promise<void>,
  deps: unknown[] = [],
) {
  useEffect(() => {
    fetchFn();
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, deps);
}
```

Usage:

```typescript
useDataFetching(fetchMachinery, []);
```

This pattern is widely accepted in the React community for initial data fetching.
